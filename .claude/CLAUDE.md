# User-Level Claude Code Configuration

## 언어 및 커뮤니케이션
- **기본 언어**: 한국어로 응답
- **기술 용어**: 영어 원문 그대로 사용 (변수명, 함수명, 라이브러리명 등)
- **응답 스타일**: 간결하고 명확하게, 불필요한 인사말이나 장황한 설명 지양
- **이모지 사용**: 명시적 요청이 없는 한 사용하지 않음

## 개발 도구 선호도
- **버전 관리**: Git 기본, 의미 있는 커밋 메시지 작성

## Git 및 커밋 규칙
- **커밋 메시지 형식**:
  ```
  <type>: <subject>

  <body> (선택)
  ```
- **Type 종류**:
  - feat: 새로운 기능 추가
  - fix: 버그 수정
  - refactor: 코드 리팩토링
  - style: 코드 포맷팅, 세미콜론 누락 등
  - docs: 문서 수정
  - test: 테스트 코드 추가/수정
  - chore: 빌드 업무, 패키지 매니저 설정 등

## 프로젝트 관리
- **TodoWrite 도구 적극 활용**: 3단계 이상의 작업은 반드시 todo 리스트로 관리
- **단계별 진행**: 한 번에 하나의 작업만 in_progress 상태 유지
- **즉시 업데이트**: 작업 완료 즉시 상태 변경

## 보안 및 개인정보
- **민감 정보 보호**:
  - 환경 변수 파일은 꼭 물어보고 변경하기
  - API 키, 토큰 등 환경변수로 관리
  - .gitignore 필수 항목 자동 추가
- **의존성 관리**:
  - 프로덕션에서는 최소한의 의존성만 사용
  - 안정적이면서 최신의 버전을 사용
  - 정기적인 보안 업데이트 확인

## 테스트
- 모든 코드는 테스트 코드를 작성

## 문서화
- **README.md 필수 항목**:
  - 프로젝트 설명
  - 설치 방법
  - 실행 방법
  - 주요 기능
  - 기술 스택
- **코드 주석**: 복잡한 로직에만 작성, 자명한 코드에는 주석 지양

## 작업 흐름
- **기능 개발**:
  1. 요구사항 명확히 파악
  2. TodoWrite로 작업 단계 정의
  3. 순차적으로 구현
  4. 각 단계 완료 후 즉시 상태 업데이트
  5. 최종 테스트 및 검증

## 금지 사항
- ❌ 백그라운드로 개발 서버 실행 금지
- ❌ GitHub CLI(gh) 사용 금지 (git 명령어 또는 웹 인터페이스 사용)
- ❌ 민감한 정보 하드코딩 금지
- ❌ 불필요한 의존성 추가 금지
- ❌ 테스트되지 않은 코드 프로덕션 배포 금지
